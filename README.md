# CoverDesign-skill

一个用于小红书单张封面设计的 Codex Skill。

它会根据用户上传的视频、文案、标题、图片素材或参考封面，判断内容类型，匹配合适的封面风格，并输出可直接用于生图工具的封面设计规范和提示词。

## 适用场景

- 分析小红书封面参考图
- 根据视频、口播稿、标题或文案生成封面提示词
- 根据人物图、产品图、食物图、家居图等素材设计单张封面
- 根据参考封面复刻设计语言，但不照抄原图
- 为不同内容匹配合适的封面类型

## 支持的封面类型

- 教程干货类：AI 教程、设计教程、运营技巧、软件技巧
- 旅行攻略类：城市攻略、路线、避坑、本地生活、旅行体验
- 生活方式类：PLOG、日常、家居、情绪表达、独处、松弛感
- 美食 PLOG 类：早餐、咖啡、烘焙、甜品、餐桌日记
- 穿搭杂志类：OOTD、Lookbook、单品推荐、风格拆解
- 直播上新类：品牌上新、直播预告、活动时间、系列发布
- 访谈综艺类：人物故事、职场话题、采访、社会观察

## 输出内容

Skill 默认输出中文内容，包括：

- 内容判断
- 推荐封面类型
- 匹配原因
- 图形构成
- 设计规范
- 单张图正向生图提示词
- 负面提示词
- 可替换标题
- 是否可以继续调用生图工具的引导语

## 生图策略

这个 Skill 是 prompt-first，不会默认直接生成图片。

如果当前环境支持生图工具，它只会先询问用户是否需要直接生成。只有用户明确回复“生成”“直接做”“调用 image”“帮我出图”等指令时，才会继续调用生图工具。

如果当前环境不支持生图，它会输出可复制的提示词，供 GPT-image、即梦、Midjourney 或其他工具使用。

## 设计规则

- 聚焦单张封面，不默认输出合集图
- 保留用户提供的可用标题
- 先分析参考图的图形构成，再迁移风格
- 不默认添加红色胶囊角标、左上角平台标签或固定小红书标签
- 提示词必须包含主体、场景、标题、排版、字体、色彩、装饰和负面约束
- 中文文字生成存在不稳定性，正式发布前建议在设计工具中覆盖关键文字

## 使用方式

将本仓库放入 Codex skills 目录后，在对话中上传素材并说明需求，例如：

```text
用 CoverDesign-skill 分析这张封面，输出单张图生图提示词。
```

```text
这是我的视频和标题，帮我匹配封面类型，并输出封面设计规范和提示词。
```

```text
根据这张参考图，给我做一个家居类小红书封面提示词。
```
